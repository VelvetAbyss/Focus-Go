# Contributing to Focus&go

Thanks for your interest in improving Focus&go.

This project follows a low-friction contribution model:

- Found a bug: open a Bug Report issue.
- Have an idea: open a Feature Request issue.
- Want to improve something quickly: open a PR directly.

## Quick Contribution Flow

1. Fork the repository.
2. Create a branch from `main`.
3. Make your change.
4. Run checks locally.
5. Open a pull request.

## Local Verification

```bash
npm install
npm run verify
```

## Pull Request Guidelines

- Keep each PR focused on one change.
- Explain what changed and why.
- Add screenshots or recordings for UI changes.
- Link the related issue when applicable.

## Code of Collaboration

- Be respectful and constructive.
- Prefer clear reproduction steps when reporting bugs.
- Keep feedback specific and actionable.

## Need Help?

Open a Feature Request or Bug Report and include enough context for maintainers to reproduce or evaluate quickly.
